# 03) Demo Walkthrough and Acceptance (Client Session)

## Demo objective
Show why Chama Station Inn should adopt this as the operational core of its future PWA.

## Suggested 20-minute walkthrough
1. **Business framing (2 min)**
   - Current site is brand-forward but operationally thin.
   - This prototype is the operations engine behind that brand.

2. **Today Operations board (4 min)**
   - Show arrivals, departures, in-house, pending check-ins/outs, overdue arrivals, contact gaps.
   - Emphasize "what staff should act on now".

3. **Upcoming Arrivals readiness (4 min)**
   - Show upcoming booked stays.
   - Show how missing guest/contact/revenue data is surfaced before arrival day.

4. **Data Quality + Action Board (4 min)**
   - Show issue detection and recommended next actions.
   - Open queue links to prove operational drill-down.

5. **Scenario switching + export (4 min)**
   - Load Balanced / Booking Rush / Data Cleanup scenarios.
   - Export CSV to show portability and reporting readiness.

6. **PWA direction close (2 min)**
   - Explain which workflows remain in WordPress and which move into PWA screens first.

## Acceptance checklist for Chama Station Inn
- [ ] Owner sees clear operational value beyond a brochure site.
- [ ] Team agrees dashboards reflect real daily operating questions.
- [ ] Team confirms vocabulary and room-theme language can be mapped.
- [ ] Team confirms this should move forward as the inn's PWA foundation.
- [ ] Team agrees first implementation sprint scope.

## Decisions to capture during session
- Required room-theme fields and naming.
- Required SOP checkpoints per stay status.
- Required permission model (owner vs desk staff).
- Required report/export cadence.
